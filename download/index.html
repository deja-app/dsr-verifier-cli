Verify your download

Always verify the checksum before running the binary.

Step 1 — Download the checksum file and its GPG signature

curl -LO https://github.com/deja-dev/dsr-verifier-cli/releases/download/v1.0.0/SHA256SUMS
curl -LO https://github.com/deja-dev/dsr-verifier-cli/releases/download/v1.0.0/SHA256SUMS.asc

Step 2 — Import the Déjà release public key (one-time setup)

gpg --keyserver keys.openpgp.org --recv-keys REPLACE_RELEASE_KEY_FINGERPRINT

Or download directly:

curl -L https://deja.dev/release-key.asc | gpg --import

Step 3 — Verify the GPG signature on SHA256SUMS

gpg --verify SHA256SUMS.asc SHA256SUMS

You should see: Good signature from "Déjà Release Key <releases@deja.dev>"

Step 4 — Verify the binary checksum

macOS / Linux:

sha256sum --check --ignore-missing SHA256SUMS

macOS (if sha256sum is not installed):

shasum -a 256 --check --ignore-missing SHA256SUMS

Step 5 — Extract and run

# macOS / Linux
tar -xzf dsr-verifier-cli-v1.0.0-darwin-arm64.tar.gz
./dsr-verifier-cli --version

# Windows (PowerShell)
Expand-Archive dsr-verifier-cli-v1.0.0-windows-amd64.zip .
.\dsr-verifier-cli.exe --version

Reproducible builds

The source code is open (github.com/deja-dev/dsr-verifier-cli). Any auditor can rebuild the binary from source and verify it matches the SHA-256 above: 

git clone https://github.com/deja-dev/dsr-verifier-cli.git
cd dsr-verifier-cli
git checkout v1.0.0

COMMIT=$(git rev-parse --short HEAD)
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
go build \
  -trimpath \
  -buildvcs=false \
  -ldflags="-s -w -X github.com/deja-dev/dsr-verifier-cli/internal/cli.BuildCommit=${COMMIT}" \
  -o dsr-verifier-cli \
  ./cmd/dsr-verifier-cli

sha256sum dsr-verifier-cli
# Compare to the linux-amd64 line in SHA256SUMS above

Full instructions and Docker-based build: BUILD.md
